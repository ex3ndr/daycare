# Monty 0.0.8 Runtime Upgrade

Upgraded `@pydantic/monty` in `packages/daycare` from `0.0.7` to `0.0.8`, refreshed Python prompt guidance for the shipped built-in modules, and added regression coverage proving `os.environ` does not leak host process environment variables into Monty execution.

## Changes
- Updated `packages/daycare/package.json` to `@pydantic/monty@0.0.8`.
- Switched the Monty Python plugin back to the package entrypoint instead of a legacy direct `index.js` path lookup.
- Updated Python execution prompt/docs to describe the supported modules: `typing`, `os`, `pathlib`, `sys`, `math`, and `re`.
- Added test coverage for `math` and `re` imports plus non-leaking `os.environ` behavior.
